# MAgent4Science

MAgent4Science is a multi-agent research steering engine for scientific
hypotheses. It helps a researcher decide whether a hypothesis is worth
pursuing before they commit months or years of work to it.

The system maps a proposed hypothesis onto the live literature, identifies
conflict and overlap, simulates how real research groups in the field may
respond, forecasts impact across multiple dimensions, mutates the hypothesis
into stronger variants, and recommends a research strategy.

Positioning: Denario can generate papers from data. MAgent4Science helps
researchers choose which paper is worth writing.

## What The System Does

Given a raw hypothesis, MAgent4Science produces:

- A structured parse of the claim, mechanism, context, population, and method.
- A literature map of nearby papers from OpenAlex and Semantic Scholar.
- Conflict and overlap analysis to show where the idea is challenged or crowded.
- A set of real research groups active in the surrounding field.
- Grounded group emulations that estimate how those groups would respond.
- Field-response scenarios, including race conditions and white-space areas.
- A six-dimensional impact forecast.
- Mutated hypothesis variants generated by explicit research operators.
- A Pareto frontier of non-dominated variants.
- A final strategy memo with a recommended direction and next actions.

## Architecture

The canonical architecture is defined in [architecture.md](architecture.md).
The target runtime is a LangGraph state machine with parallel branches wherever
possible.

```text
Raw hypothesis
  -> Parser
  -> Cartographer + Conflict Detector + Overlap Auditor
  -> Group Identifier
  -> Group Emulators + Groundedness Checker
  -> Trajectory Synthesiser
  -> Impact Forecaster
  -> Mutator
  -> Variant Re-scoring
  -> Pareto Curator
  -> Strategist
  -> Streamlit dashboard
```

The architecture is designed around 12 agent roles and roughly 20 agent
instances per run once the research-group emulators fan out.

| Agent | Role |
| --- | --- |
| Parser | Converts free-text hypotheses into structured fields. |
| Cartographer | Retrieves and clusters nearby literature. |
| Conflict Detector | Finds papers that contradict or weaken the hypothesis. |
| Overlap Auditor | Measures how crowded the idea already is. |
| Group Identifier | Finds active research groups in the literature neighbourhood. |
| Group Emulator | Simulates grounded responses from each research group. |
| Groundedness Checker | Verifies emulator outputs against each group's paper history. |
| Trajectory Synthesiser | Turns group responses into field-response scenarios. |
| Impact Forecaster | Predicts impact across six dimensions. |
| Mutator | Generates alternative versions of the hypothesis. |
| Pareto Curator | Selects variants that are not dominated across impact dimensions. |
| Strategist | Produces the final recommendation and research memo. |

## Impact Model

MAgent4Science deliberately avoids collapsing scientific impact into a single
citation score. Each hypothesis and variant is scored across six dimensions:

| Dimension | Meaning |
| --- | --- |
| Volume | Total expected attention. |
| Velocity | Speed of recognition. |
| Reach | Cross-disciplinary spread. |
| Depth | Foundational value versus incremental value. |
| Disruption | Whether the work may displace prior assumptions. |
| Translation | Real-world uptake in clinical, patent, or policy contexts. |

## Hypothesis Mutation

The Mutator creates variants using seven explicit operators:

- Generalise: broaden the population or setting.
- Narrow: focus on a sharper context or subgroup.
- Substitute mechanism: keep the outcome but change the proposed pathway.
- Shift scale: move between acute and chronic, or in vitro and in vivo.
- Cross-pollinate: import a method from an adjacent field.
- Invert: test the null or opposite claim directly.
- Combine: fuse the hypothesis with an adjacent open question.

Each variant is re-scored by the impact forecaster, then the Pareto Curator
selects variants that improve the trade-off surface.

## Validation

The planned validation harness backtests the system on historical papers:

- Sample papers from 2018 across computer science, biology, and materials.
- Restrict all retrieval and agent context to pre-publication information.
- Predict the six impact dimensions.
- Compare predictions with 2024 outcomes.
- Report Spearman correlations against simple baselines.

This date-filtering constraint is central: the backtest must not allow future
information to leak into the prediction run.

## Dashboard

The demo interface is planned as a single-page Streamlit dashboard with:

- A hypothesis input and run control.
- A blooming literature graph centred on the user's hypothesis.
- Research-group portraits with interest scores and proposed directions.
- A six-dimension radar chart.
- A Pareto scatter plot for original and mutated hypotheses.
- A strategy memo panel.
- A persistent predicted-versus-actual backtest scatter plot.

## Technical Stack

| Layer | Target choice |
| --- | --- |
| Orchestration | LangGraph |
| LLM provider | OpenAI GPT-5 family, routed per agent |
| Tracing | Langfuse |
| Literature data | OpenAlex and Semantic Scholar |
| Cache | Local SQLite keyed by query hash |
| Dashboard | Streamlit, Plotly, cytoscape.js |
| Schemas | Pydantic |

The model routing is intended to be provider-agnostic. Lightweight extraction
and verification agents use smaller models, grounded persona and forecasting
agents use mid-tier models, and synthesis or strategy agents use stronger
reasoning models.

## Repository Status

This repository currently contains the project architecture, task plan, early
agent prompt scaffolds, and a Semantic Scholar integration. The architecture in
[architecture.md](architecture.md) is the canonical direction; the earlier
agent notes in [AgentsIdeas.md](AgentsIdeas.md) and `agents/` are useful
scaffolding but do not supersede the 12-agent pipeline.

Immediate implementation targets from [team_tasks.md](team_tasks.md):

- `schemas.py`: shared Pydantic contracts for all agents.
- `llm_client.py`: model routing, structured output, and Langfuse tracing.
- `run.py`: CLI entry point for end-to-end LangGraph execution.
- `agents/*.py`: concrete implementations of the 12 agent roles.
- `tools/`: OpenAlex and Semantic Scholar retrieval with SQLite caching.
- `app/`: Streamlit dashboard.
- `scripts/backtest.py`: historical validation harness.
